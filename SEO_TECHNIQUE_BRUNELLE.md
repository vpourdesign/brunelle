## 4. Gabarit SEO par type de page

  2. Les quartiers de Blainville (liens internes vers les 9 pages quartier)

- **Internal linking** : au moins 8 liens sortants vers pages quartier et types
- **Images** : photos originales de Blainville (parc Fontainebleau, mairie, etc.) + `alt` descriptif

## 5. SEO technique

### 5.1  Stack et performance
- **Framework** : React + Vite + TypeScript (déjà prévu).
- **Rendu** : **SSG** ou **SSR** obligatoire (pas SPA pur) →
  utiliser **Next.js 15** ou **Astro 4** pour que Google voie le
  contenu sans JS.
  - Recommandation : **Astro** (meilleur pour contenu statique +
    îlots interactifs + vitesse supérieure à Next SSG).
- **Core Web Vitals cibles** :
  - LCP < 1.8 s
  - INP < 200 ms
  - CLS < 0.05
- **Images** : `<Image>` avec `loading="lazy"` sauf LCP,
  formats AVIF + WebP, tailles responsive.
- **Fonts** : auto-hébergées, `font-display: swap`,
  preload du fichier principal.
- **Budget JS** : < 150 Ko sur l'accueil.

### 5.2  Indexation
- `robots.txt` ouvert, pointe vers `sitemap.xml`.
- `sitemap.xml` généré au build (+ regen quotidien pour les fiches Centris).
  Segmenter : `sitemap-pages.xml`, `sitemap-quartiers.xml`,
  `sitemap-proprietes.xml`, `sitemap-blog.xml`.
- `canonical` systématique sur chaque page.
- **Hreflang** : `fr-CA` + `en-CA` sur toutes les pages bilingues.
- Éviter le contenu dupliqué (paramètres de filtre → `noindex` ou
  canonical vers la version propre).

### 5.3  Données structurées (JSON-LD)
À injecter en `<script type="application/ld+json">` :

- **Global (toutes pages)** :
  - `Organization` / `RealEstateAgent` avec
    `name`, `logo`, `image`, `url`, `telephone`,
    `address` (PostalAddress), `geo`, `openingHours`,
    `sameAs` (Facebook, Instagram, LinkedIn, YouTube),
    `aggregateRating` (une fois Google Reviews connecté).
  - `BreadcrumbList`.
- **Page-ville/quartier** : `Place` + `FAQPage`.
- **Fiche propriété** : `RealEstateListing` + `Residence`.
- **Article** : `Article` + `Person` (author).
- **Témoignages** : `Review` + `AggregateRating`.

### 5.4  Local SEO (crucial pour battre Michel Salomon)
1. **Google Business Profile** dédié à Alain Brunelle —
   catégorie principale *Real estate agent*, catégories secondaires
   *Real estate agency*, *Real estate consultant*.
   Remplir à 100 % : photos, vidéos, services, horaires, zones
   desservies, produits (propriétés vedettes).
2. **NAP cohérent partout** (Nom + Adresse + Téléphone identiques
   sur : site, GBP, Centris, RE/MAX, annuaires).
3. **Citations locales** à créer :
   Centris, YellowPages.ca, PagesJaunes, 411.ca, Canada411,
   Houzz, Realtor.ca, Yelp, Foursquare, Facebook, LinkedIn,
   Chambre de commerce Thérèse-De Blainville.
4. **Backlinks locaux** à viser :
   - Ville de Blainville (sponsor d'événement / lien presse)
   - Hebdos Rive-Nord (Journal Nord-Info)
   - Écoles et clubs sportifs locaux
   - Partenaires (notaires, inspecteurs, déménageurs)
5. **Reviews Google** : viser 50+ avis 5★ en 12 mois,
   demande automatisée post-transaction (SMS + courriel).

### 5.5  Sécurité / base
- HTTPS partout, HSTS, redirection www → apex (ou inverse, choisir une).
- Certificat SSL Let's Encrypt auto-renouvelé.
- Headers : `Content-Security-Policy`, `X-Content-Type-Options`,
  `Referrer-Policy`, `Permissions-Policy`.

## 7. Intégration flux Centris (rappel — cf. STRUCTURE_CENTRIS.md)
Pour le SEO, l'important :
- **1 URL par propriété** (slug lisible).
- Régénération quotidienne du sitemap propriétés.
- Quand une propriété est **vendue** → la page reste en ligne,
  marquée « Vendue », avec prix affiché (ou masqué selon préférence).
  Cela préserve le jus SEO et sert de preuve sociale.
- Ne JAMAIS supprimer une URL de propriété (301 vers la nouvelle
  version si re-listée, ou conservation telle quelle).
